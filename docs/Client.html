Class: Client

Client(token, isBot)

Extends the Discord class but will also create and maintain a connection to the gateway, allowing some extra functions.

Constructor

new Client(token, isBot)

Create a connection to the gateway.
Parameters:
Name Type Default Description
token string The secret_key to use when interfacing with the API.
isBot boolean true Is the secret key associated with a bot account or not.
Properties:
Name Type Description
isBot boolean Is the token associated with this client a bot or not. This affects the authorization headers and gateway endpoints.
shardCount number The number of shards received when requesting the gateway URL.
socket Object The websocket used to communicate with the gateway.
me Object The object containing the current user. This will be set when the 'ready' dispatch is received.
heartbeatIntervalObj Object The object returned by the setInterval() call for the hearbeat function. Needed to stop the heartbeat function being called after the connection is closed.
lastSequenceNum number The last sequence number received from the gateway. Needed for resuming a session and heartbeats.
hearbeatAcknowledged boolean Whether or not the last heartbeat was acknowledged by the server.
sessionID string The session ID received from the server that can be used to resume the session when reconnecting.
resuming boolean Is the client currently attempting to resume the connection. A few functions will behave differently if they're connecting for the first time or attempting a resume.
dispatchCallbacks Object The object that stores all callbacks resgistered to certain dispatch events. The keys are the event names and the values are the callbacks.
commandChar string The character that needs to be the first character in a message for it to be considered a command.
commandCallbacks Object The object that stores all callbacks registered to different command words. The keys are the command words and the values are the callbacks.
guilds Object All guilds that the bot is in. These need to be stored because when requesting a guild from the API, you don't get all the information that you do in a GUILD_CREATE event.
GatewayOpcodes Object All the opcodes that go through the gateway and their meanings.

bulkDeleteMessages(channelID, messages) → {Promise}

Delete multiple messages in a single request. This endpoint can only be used on guild channels and requires the 'MANAGE_MESSAGES' permission. Returns a 204 empty response on success. Fires multiple Message Delete Gateway events. Any message IDs given that do not exist or are invalid will count towards the minimum and maximum message count (currently 2 and 100 respectively). Additionally, duplicated IDs will only be counted once.

createMessage(channelID, content)

Post a message to a guild text or DM channel. If operating on a guild channel, this endpoint requires the 'SEND_MESSAGES' permission to be present on the current user. If the tts field is set to true, the SEND_TTS_MESSAGES permission is required for the message to be spoken. Returns a message object. Fires a Message Create Gateway event. The maximum request size when sending a message is 8MB.

Name Type Description
